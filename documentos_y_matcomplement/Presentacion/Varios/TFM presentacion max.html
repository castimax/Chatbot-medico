Presentación de los Cambios Implementados en el TFM

Diapositiva 1: Título

Desarrollo de un Sistema RAG para Consultorio de Datos de Salud y Médicos

Diapositiva 2: Justificación del Proyecto

Motivación:
Los avances en inteligencia artificial permiten la personalización de la salud. Este proyecto busca desarrollar un algoritmo predictivo basado en datos genómicos y biométricos.

Problema a Resolver:
La falta de recomendaciones personalizadas basadas en datos genómicos y biométricos.

Diapositiva 3: Arquitectura del Sistema

main.py:
Configuración del modelo y carga de documentos.

custom_agent.py:
Definición del agente personalizado para manejar consultas.

streamlit_app.py:
Interfaz de usuario con Streamlit.

Diapositiva 4: Configuración Inicial y Librerías (main.py)

  1. Importación de librerías esenciales:
    Se importan las librerías necesarias para el funcionamiento del sistema.

  2. Configuración de variables de entorno y caché:
    Se establecen las variables de entorno y se configura el uso de caché para mejorar la eficiencia.

  3. Inicialización de FastAPI:
    Se inicializa FastAPI para crear la API del servicio.

Diapositiva 5: Configuración del Modelo y Embeddings (main.py)

Carga de la clave API:
Se carga la clave API necesaria para el modelo de lenguaje ChatGroq.

Preparación de embeddings:
Se construye un índice vectorial usando FAISS para los embeddings del modelo.

Diapositiva 6: Funciones Clave y Endpoints (main.py)

  1. Endpoint /ask:
    Se define el endpoint /ask para manejar las preguntas de los usuarios.

  2. Uso de caché:
    Se utiliza caché para almacenar las respuestas y mejorar la eficiencia.

  3. Carga y procesamiento de documentos:
    Se cargan y procesan documentos desde diversas fuentes para alimentar el sistema.

Diapositiva 7: Definición del Agente Personalizado (custom_agent.py)

Clase CustomAgent:
Se define la clase CustomAgent que hereda de LLMSingleActionAgent.

Métodos input_keys y plan:
Se implementan los métodos input_keys y plan para planificar acciones basadas en pasos intermedios.

Diapositiva 8: Parser de Salida Personalizado (custom_agent.py)

CustomOutputParser:
Se define la clase CustomOutputParser que analiza la salida del modelo de lenguaje.

Decidir siguiente acción:
Se implementan métodos para decidir si la respuesta es final o si se necesitan más acciones.

Diapositiva 9: Creación del Agente y Ejecutor (custom_agent.py)

create_custom_tools_agent:
Se define la función para configurar el agente personalizado integrando el modelo de lenguaje con herramientas externas.

Preparación de prompts:
Se preparan los prompts y se configura el ejecutor para el agente personalizado.

Diapositiva 10: Nuevas Funcionalidades Implementadas

Web Scraping con BeautifulSoup:
Se añade la capacidad de hacer scraping en sitios web especializados como la Clínica Mayo para obtener información adicional sobre enfermedades.

API de ClinVar:
Se integra la API de ClinVar para obtener información genética específica sobre enfermedades.

Traducción Automática:
Las consultas se traducen al inglés antes de realizar búsquedas y se devuelven las respuestas traducidas al idioma original del usuario.

Almacenamiento de Datos:
Se guarda la información obtenida en un archivo JSON ubicado en el directorio data/ para futuras consultas y procesamiento.

Diapositiva 11: Interfaz de Usuario con Streamlit (streamlit_app.py)

Configuración de la interfaz:
Se configura la interfaz de usuario utilizando Streamlit.

Entrada de texto:
Se agrega un campo de entrada de texto para que el usuario ingrese preguntas.

Botón de envío:
Se incluye un botón para que el usuario pueda enviar las preguntas al backend.

Diapositiva 12: Diagrama de Arquitectura

Diagrama Mermaid:
Se muestra el diagrama de arquitectura del sistema, detallando el flujo de datos y la interacción entre componentes.

El foco está en destacar la integración de múltiples fuentes de datos y la capacidad del bot para manejar consultas complejas y específicas sobre información genética.
